基于 shake_hands 握手数据集的实例解析

v3.0 格式 SO-101 机械臂 30 集 · 10113 帧 · 30fps

2info.json — 数据集定义

描述数据集的 schema：有哪些特征、每个特征的类型和维度。

{ "codebase_version": "v3.0", "robot_type": "so101_follower", "total_episodes": 30, "total_frames": 10113, "fps": 30, "features": { "action": { "dtype": "float32", "shape": [6], "names": ["shoulder_pan.pos", "shoulder_lift.pos", "elbow_flex.pos", "wrist_flex.pos", "wrist_roll.pos", "gripper.pos"] }, "observation.state": { "dtype": "float32", "shape": [6], "names": ["shoulder_pan.pos", ...] }, "observation.images.front": { "dtype": "video", "shape": [1080, 1920, 3] }, "timestamp": { "dtype": "float32", "shape": [1] }, // 自动添加 "frame_index": { "dtype": "int64", "shape": [1] }, // 自动添加 "episode_index": { "dtype": "int64", "shape": [1] }, // 自动添加 "index": { "dtype": "int64", "shape": [1] }, // 自动添加 "task_index": { "dtype": "int64", "shape": [1] } // 自动添加 } }
用户只需定义 3 个特征：action、observation.state、observation.images.front。 后 5 个索引列是框架自动添加的，无需手动指定。
特征dtypeshape含义

observation.images.frontvideo(1080, 1920, 3)前置摄像头图像，存储在 mp4 文件中

3data/ — Parquet 核心数据

Parquet 是列式存储格式，每行是一帧，每列是一个特征。10113 行 × 7 列。

什么是列式存储？

CSV 按行存：一行数据连续放。Parquet 按列存：同一列的数据连续放。

// CSV（行式） row0: [-0.93,-99.58,...], [-1.65,-99.16,...], 0.0 row1: [-0.93,-99.58,...], [-1.65,-99.16,...], 0.033 // Parquet（列式） action列: [-0.93,-99.58,...] [-0.93,-99.58,...] ... observation.state列: [-1.65,-99.16,...] [-1.65,-99.16,...] ... timestamp列: 0.0 0.033 0.067 ...
Parquet 的优势
✓ 压缩率高：同列数据类型相同、值域相近，压缩 3~10 倍
✓ 按列读取：pd.read_parquet(f, columns=["action"]) 只加载需要的列
✓ 类型自描述：schema 内嵌文件头，不依赖外部定义
✓ 支持复杂类型：action 列存的是 list[float] 变长数组

5stats.json — 训练归一化的关键输入

不是可选的统计文件，而是训练时数据归一化的必要参数来源。

每个策略都配置了归一化方式
// ACT 策略的归一化配置 normalization_mapping = { "VISUAL": IDENTITY, // 图像不归一化 "STATE": MIN_MAX, // 关节状态 → [-1, 1] "ACTION": MIN_MAX, // 动作 → [-1, 1] } // SmolVLA / Pi0 策略 normalization_mapping = { "STATE": MEAN_STD, // 标准化: (x-mean)/std "ACTION": MEAN_STD, }
5 种归一化模式
模式公式使用策略
MIN_MAX2(x-min)/(max-min)-1ACT, VQ-BeT, TDMPC
MEAN_STD(x-mean)/stdSmolVLA, Pi0, Wall-X
QUANTILES2(x-q01)/(q99-q01)-1Pi0.5
QUANTILE10基于 q10/q90—
IDENTITY不处理图像、语言

stats.json 中的归一化参数

observation.state 的统计量
{ "observation.state": { "min": [-19.62, -99.33, 8.03, -3.64, -5.05, 1.20], "max": [ 2.02, 11.40, 99.27, 78.66, 14.14, 36.79], "mean": [ -6.57, -68.43, 73.25, 45.96, 2.21, 10.20], "std": [ 3.82, 34.90, 25.52, 18.92, 3.29, 9.10], "q01": [-11.46, -99.26, 32.59, 17.48, -0.85, 1.78], "q99": [ -2.89, -14.76, 97.91, 64.81, 6.44, 27.63] } }

MIN_MAX 归一化实例

shoulder_lift（第 2 维）：原始值 [-99.33, 11.40] → 归一化到 [-1, 1]
原始: -99.33
→ -1.00
原始: -68.43
→ -0.44
原始: -40.00
→ +0.07
原始: 11.40
→ +1.00
// 归一化公式 normalized = 2 * (x - min) / (max - min) - 1 // 反归一化（推理时） original = (normalized + 1) / 2 * (max - min) + min
为什么必须归一化？ 各关节数值范围差异大：shoulder_lift 在 -99~11，elbow_flex 在 8~99。 不做归一化的话，梯度会被大数值主导，训练不稳定。

7tasks.parquet — 任务描述

索引（任务文本）task_index

所有帧的 task_index=0 都指向这个任务。这是自然语言任务描述，供 SmolVLA 等 VLA 模型使用。 ACT 不使用任务文本。

8ACT 训练时如何读取数据

ACT 通过 delta_timestamps 机制从数据集中提取"当前观测 + 未来动作序列"。

ACT 的 delta_indices 配置
维度值含义
observation_delta_indicesNone观测只取当前帧，不取历史帧
action_delta_indices[0, 1, ..., 99]动作取未来 100 帧（chunk_size）
reward_delta_indicesNone不使用奖励（模仿学习）

__getitem__(t=50) 返回的 batch 字典

{ // ---- 用户定义的特征 ---- "observation.state": tensor shape (6,) // 第 50 帧的关节状态 "observation.images.front": tensor shape (3, 1080, 1920) // 第 50 帧的图像 "action": tensor shape (100, 6) // 第 50~149 帧的动作 "action_is_pad": tensor shape (100,) // 超出 episode 边界处为 True // ---- 自动添加的索引 ---- "index": scalar // 全局帧序号 "episode_index": scalar // episode ID "frame_index": scalar // 集内帧序号 "timestamp": scalar // 集内时间 "task_index": scalar // 任务 ID "task": "Shanke Hands" // 任务文本 }

action_is_pad 的作用

loss 屏蔽：当 t=50、episode 在 t=349 结束时，action chunk 的第 300~349 帧是真实数据，第 350~149 帧（对应 episode 外的部分）用零填充， action_is_pad 对应位置为 True。ACT 的 L1 loss 会屏蔽这些位置，不参与梯度计算。

ACT forward() 读取的 key

Batch KeyShape用途
observation.state(B, 6)送入 VAE encoder 和 transformer encoder
observation.images.front(B, 3, H, W)送入 ResNet backbone 提取视觉特征
action(B, 100, 6)VAE encoder 的目标 + L1 loss 计算
action_is_pad(B, 100)loss mask，屏蔽 episode 边界外的帧

10关键总结

统一格式
  • 所有策略（ACT、Diffusion、TDMPC、SmolVLA...）共享同一个 LeRobotDataset
  • 差异仅在 delta_timestamps（取多大时间窗口）和归一化方式
  • 新算法只需定义自己的 delta_indices 和 processor 即可
三个关键文件
  • info.json：schema 定义（有什么特征、什么类型）
  • data/*.parquet：每帧的 state/action 等结构化数据
  • stats.json：训练归一化参数（不是可选的统计文件）
